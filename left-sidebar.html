Digital Forensics

Digital Evidence Handling & Crime Scene Processing

Project Summary

This project focuses on a digital forensic investigation of a suspected insider data breach in a corporate environment. The case involved a powered-on workstation, a connected SanDisk 32GB USB device, deleted client database evidence, suspicious email communication, and possible cloud-based data exfiltration through Dropbox and Google Drive.

View Digital Forensics PDF

What I Did

Identified Evidence

I identified important digital evidence such as system logs, USB logs, browser history, suspicious email, deleted files, RAM, and network traffic.

Followed Volatility Order

I prioritized volatile evidence such as live network traffic and RAM before collecting non-volatile storage evidence.

Recovered Deleted Evidence

I analyzed forensic artifacts and recovered deleted data such as client_database_deleted.csv using Autopsy.

Correlated Findings

I connected system login activity, USB insertion, suspicious email, and browser history to build a clear timeline of the incident.

PENETRATION TESTING

NETWORK TRAFFIC CAPTURE AND ANALYSIS USING WIRESHARK

Project Summary

This project focuses on network traffic capture and analysis using Wireshark. The analysis involved inspecting protocol layers, understanding TCP communication, examining packet details, and identifying how data is transmitted across a network.

Through this project, I explored how Wireshark can reveal important security information such as FTP credentials, HTTP login attempts, successful and failed authentication attempts, and weak authentication methods such as HTTP Basic Authentication.

View Wireshark Analysis PDF

What I Learned From This Case Study

Packet Inspection

I learned how to inspect packet list, packet details, and packet bytes to understand how network communication is structured.

Protocol Layer Analysis

I explored how Wireshark displays OSI Layer 1 to Layer 4 information, including frame details, MAC addresses, IP addresses, ports, and TCP flags.

Credential Exposure

I learned how unencrypted protocols such as FTP and HTTP can expose usernames and passwords in readable form.

Authentication Weakness

I understood why HTTP Basic Authentication is weak without HTTPS because Base64 encoding can be decoded and is not proper encryption.
